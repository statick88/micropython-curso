Manejo de Errores: Cuando las Cosas No Salen como Esperamos

La realidad de la programación: Los errores pasan

En robótica educativa, los errores son inevitables: - Un sensor que no responde - Una lectura de distancia que da un valor negativo imposible - Un motor que intenta girar más de lo que físicamente puede - Una conexión que se pierde temporalmente

En lugar de que nuestro programa falle catastróficamente, Python nos permite manejar errores de manera elegante.

Analogía del mundo real: El sistema de emergencia del robot

Imagina un robot de exploración espacial. Si algo sale mal: - El sistema no se destruye, sino que activa un “plan de contingencia” - Registra qué salió mal para que los ingenieros lo analicen después - Intenta una solución alternativa si es posible - Si no hay solución, se pone en modo seguro para no dañarse

Esto es exactamente lo que hacemos con el manejo de errores en Python.

Tipos comunes de errores

1. Errores de sintaxis

El código no sigue las reglas de Python:

# ❌ Error: falta paréntesis
print "Hola mundo"

# ✅ Correcto:
print("Hola mundo")

2. Errores de tipo

Usamos el tipo de dato wrong:

# ❌ Error: no podemos sumar texto + número
mensaje = "La distancia es"
distancia = 10
print(mensaje + distancia)  # TypeError

# ✅ Correcto: convertir a string o usar f-string
print(mensaje + str(distancia))
print(f"La distancia es {distancia}")

3. Errores de índice

Accedemos a una posición que no existe:

# ❌ Error: la lista solo tiene 3 elementos (índices 0, 1, 2)
colores = ["rojo", "verde", "azul"]
print(colores[3])  # IndexError: list index out of range

# ✅ Correcto:
print(colores[2])  # Accede al último

4. Errores de valor

El valor está fuera del rango válido:

# ❌ Error: no podemos convertir "hola" a número
numero = int("hola")  # ValueError

# ✅ Correcto: validar antes
valor = "50"
if valor.isdigit():
    numero = int(valor)

La estructura try-except

Python usa try (intentar) y except (excepto) para manejar errores:

# Estructura básica
try:
    # Código que podría fallar
    resultado = 10 / 0
except ZeroDivisionError:
    # Qué hacer si ocurre este error específico
    print("⚠️ No se puede dividir por cero")

Ejemplo práctico: Lectura de sensor

def leer_sensor_distancia():
    """
    Simula la lectura de un sensor de distancia.
    En un caso real, esto se conectaría al hardware.
    """
    import random
    
    # Simulamos diferentes escenarios
    escenario = random.choice(["normal", "error_conexion", "valor_invalido"])
    
    if escenario == "normal":
        return random.uniform(5, 100)  # Distancia normal
    elif escenario == "error_conexion":
        raise ConnectionError("Sensor no responde")
    else:
        raise ValueError("Lectura fuera de rango válido")

# Manejo robusto de la lectura
for intento in range(3):
    try:
        distancia = leer_sensor_distancia()
        print(f"Distancia medida: {distancia:.1f} cm")
        break  # Salir del bucle si todo sale bien
        
    except ConnectionError as e:
        print(f"⚠️ Error de conexión (intento {intento + 1}/3): {e}")
        if intento < 2:
            print("   Reintentando en 1 segundo...")
        else:
            print("   ❌ Sensor no disponible después de 3 intentos")
            
    except ValueError as e:
        print(f"⚠️ Valor inválido: {e}")
        print("   Saltando esta lectura...")
        
    except Exception as e:
        # Captura cualquier otro error no previsto
        print(f"❌ Error inesperado: {type(e).__name__}: {e}")
        break

Manejo de múltiples errores

Podemos manejar diferentes tipos de error de formas distintas:

def controlar_motor(velocidad, duracion):
    """
    Controla un motor con validación de parámetros.
    """
    try:
        # Validar velocidad
        if not isinstance(velocidad, (int, float)):
            raise TypeError(f"Velocidad debe ser número, no {type(velocidad).__name__}")
        
        if velocidad < -100 or velocidad > 100:
            raise ValueError(f"Velocidad {velocidad} fuera de rango (-100 a 100)")
        
        # Validar duración
        if duracion <= 0:
            raise ValueError("La duración debe ser positiva")
        
        # Simular movimiento
        print(f"🏃 Motor a {velocidad} rpm por {duracion} segundos")
        return True
        
    except TypeError as e:
        print(f"❌ Tipo de dato incorrecto: {e}")
        return False
        
    except ValueError as e:
        print(f"❌ Valor fuera de rango: {e}")
        return False
        
    except Exception as e:
        print(f"❌ Error inesperado: {e}")
        return False

# Pruebas
print("--- Prueba 1: valores válidos ---")
controlar_motor(50, 2)

print("\n--- Prueba 2: velocidad fuera de rango ---")
controlar_motor(150, 2)

print("\n--- Prueba 3: duración negativa ---")
controlar_motor(50, -1)

print("\n--- Prueba 4: tipo incorrecto ---")
controlar_motor("rápido", 2)

El bloque finally

El código en finally siempre se ejecuta, sin importar si hubo error o no:

def ejecutar_mision():
    """Simula una misión de robot con recursos que deben limpiarse."""
    print("🚀 Iniciando misión...")
    
    try:
        print("  📡 Conectando con servidor...")
        # Simular que algo sale mal
        raise RuntimeError("Pérdida de conexión")
        
    except RuntimeError as e:
        print(f"  ⚠️ Error: {e}")
        
    finally:
        # Esto siempre se ejecuta -cleanup de recursos
        print("  🧹 Limpiando recursos...")
        print("  🔌 Desconectando sensores...")
        print("  💤 Entrando en modo bajo consumo")

ejecutar_mision()

raise: Lanzar errores manualmente

A veces necesitamos lanzar nuestros propios errores:

def configurar_sensor(sensor_id, modo):
    """
    Configura un sensor con validación.
    """
    # Validar ID del sensor
    sensores_validos = [1, 2, 3, 4, 5]
    if sensor_id not in sensores_validos:
        raise ValueError(f"Sensor ID {sensor_id} no válido. Use: {sensores_validos}")
    
    # Validar modo
    modos_validos = ["lectura", "calibracion", "apagado"]
    if modo not in modos_validos:
        raise ValueError(f"Modo '{modo}' no válido. Use: {modos_validos}")
    
    print(f"✅ Sensor {sensor_id} configurado en modo '{modo}'")
    return True

# Pruebas
try:
    configurar_sensor(3, "lectura")
    configurar_sensor(99, "lectura")  # Esto fallará
except ValueError as e:
    print(f"Error: {e}")

Aserciones: Verificaciones rápidas

Las assertions son verificaciones rápidas que podemos usar durante el desarrollo:

def calcular_velocidad(distancia, tiempo):
    """Calcula velocidad media."""
    assert tiempo > 0, "El tiempo debe ser positivo"
    assert distancia >= 0, "La distancia no puede ser negativa"
    
    velocidad = distancia / tiempo
    return velocidad

# Uso normal
v = calcular_velocidad(100, 10)
print(f"Velocidad: {v} m/s")

# Esto causaría error de aserción:
# v = calcular_velocidad(100, 0)  # AssertionError: El tiempo debe ser positivo

Nota: Las assertions se pueden deshabilitar con python -O, así que no confíes en ellas para validación de producción.

Mini-desafío: Sistema de seguridad del robot

Crea un sistema que: 1. Intente leer el sensor de temperatura 2. Si la lectura es menor a 0 o mayor a 100, lance un error personalizado 3. Si hay error de conexión, reintente hasta 3 veces 4. Siempre muestre el estado final del sistema

Posible solución 
class TemperaturaInvalidaError(Exception):
    """Error personalizado para temperaturas fuera de rango."""
    pass

def leer_temperatura():
    """Simula lectura de sensor de temperatura."""
    import random
    valor = random.choice([15, 25, -5, 150, 35])  # Algunos inválidos
    return valor

def verificar_temperatura(temp):
    """Verifica que la temperatura sea válida."""
    if temp < 0 or temp > 100:
        raise TemperaturaInvalidaError(f"Temperatura {temp}°C fuera de rango válido (0-100)")
    return True

# Sistema principal
print("=== Sistema de Monitoreo de Temperatura ===\n")

for intento in range(3):
    try:
        print(f"Intento {intento + 1}: Leyendo sensor...")
        temperatura = leer_temperatura()
        
        verificar_temperatura(temperatura)
        
        print(f"✅ Temperatura válida: {temperatura}°C")
        print("Sistema operando normalmente")
        break
        
    except TemperaturaInvalidaError as e:
        print(f"⚠️ {e}")
        print("   Requiere intervención humana")
        break
        
    except ConnectionError as e:
        print(f"⚠️ Error de conexión: {e}")
        if intento < 2:
            print(f"   Reintentando... ({intento + 1}/3)")
        else:
            print("❌ Sensor no disponible después de 3 intentos")
            
    finally:
        print("   [Log de eventos actualizado]\n")

Buenas prácticas de manejo de errores

✅ Haz esto:

  • Sé específico con los tipos de error que capturas
  • Siempre incluye logging o mensaje cuando occurs un error
  • Usa finally para cleanup de recursos
  • Crea errores personalizados para tu dominio (robótica)
  • Considera qué debe hacer el programa si hay error (reintentar, modo seguro, etc.)

❌ Evita esto:

  • Capturar Exception indiscriminadamente (ocultas errores reales)
  • No hacer nada en el except (silenciar errores)
  • Usar assertions para validación de producción
  • Confiar ciegamente en que “nunca occurirá este error”
