Laboratorio 1.1: “Hola Mundo” en Pybricks

⚡ Nivel: Principiante | ⏱️ Tiempo: 25 minutos

Objetivo del Laboratorio

Validar que el entorno de desarrollo está correctamente configurado y que podemos comunicar instrucciones al hardware físico del Hub LEGO SPIKE.

Materiales Requeridos

Componente Especificación
Hub LEGO SPIKE Prime o Essential
Cable USB original LEGO (no genérico)
PC/Laptop Con navegador Chrome/Edge
Conexión Internet para code.pybricks.com

⚡ Procedimiento Paso a Paso

Paso 1: Conexión Física (5 minutos)

# En tu terminal, verifica los dispositivos conectados

# Linux (Ubuntu/Debian)
ls -la /dev/tty* | grep -E "(USB|ACM)"

# macOS
ls -la /dev/tty.* | grep usb

# Windows (PowerShell)
Get-ChildItem -Path COM*

Output Esperado:

# Linux
crw-rw---- 1 root root 188, 0 Jan 15 10:30 /dev/ttyUSB0

# macOS
crw-rw---- 1 root root 21, 1 Jan 15 10:30 /dev/tty.usbserial-1420

⚠️ Si no ves ningún dispositivo: El cable no es de datos. Usa el cable original LEGO.

Paso 2: Acceso al IDE (2 minutos)

  1. Abre Chrome o Edge (NO Firefox ni Safari)
  2. Navega a: https://code.pybricks.com/
  3. Conecta el Hub con el cable USB
  4. Presiona el botón de encendido del Hub (botón central)

Deberías ver en el IDE:

┌─────────────────────────────────────┐
│  Pybricks IDE                       │
│  ┌───────────────────────────────┐ │
│  │ 🟢 Hub Connected (Prime)      │ │
│  │ Firmware: 2.0.0               │ │
│  └───────────────────────────────┘ │
└─────────────────────────────────────┘

Paso 3: Tu Primer Programa (10 minutos)

Copia y pega este código en el editor:

1from pybricks.hubs import PrimeHub
2from pybricks.parameters import Color
3from pybricks.tools import wait

4hub = PrimeHub()

def hola_stark():
    """Protocolo de validación de sistemas."""
5    hub.light.on(Color.GREEN)
6    hub.display.text("HOLA")
7    hub.speaker.beep()
8    wait(2000)
9    hub.light.off()

if __name__ == "__main__":
10    hola_stark()
1
Importamos el núcleo del Hub LEGO SPIKE.
2
Cargamos el set de parámetros para gestión de colores.
3
Herramienta para pausas precisas en la ejecución.
4
Instanciamos el cerebro del robot.
5
Activamos el LED central en verde - indica sistema nominal.
6
Enviamos texto a la matriz de 5x5 píxeles.
7
El buzzer emite un tono de confirmación.
8
Pausa de seguridad antes de apagar.
9
Apagamos el LED - operación completada.
10
Punto de entrada del script.

Paso 4: Ejecución (3 minutos)

  1. Presiona el botón “Run” (▶️) en el IDE
  2. Observa el comportamiento del Hub

Output Esperado en Consola:

>>> Running script.py
>>> HOLA
>>> 🔊 Beep executed
>>> Script completed successfully

Comportamiento Físico: - 💡 LED central se ilumina en VERDE - 📺 Matriz muestra la palabra “HOLA” - 🔊 Sonido “beep” audible - ⏸️ 2 segundos después, LED se apaga

Paso 5: Personalización (5 minutos)

Modifica el código para mostrar tu nombre:

# Cambia esta línea:
hub.display.text("HOLA")

# Por tu nombre (máximo 5 letras):
hub.display.text("DIEGO")  # Ejemplo

Desafío Extra: Cambia el color a AZUL y el mensaje a “READY”

🛠️ Centro de Diagnóstico

Error Síntoma Causa Solución
“Hub not found” IDE no detecta Hub Cable incorrecto Usar cable LEGO original
“Firmware outdated” Mensaje amarillo en IDE Pybricks requiere update Click “Update Firmware”
“Permission denied” Error en terminal Linux Sin acceso a tty sudo chmod 666 /dev/ttyUSB0
“Script timeout” LED parpadea rojo Código en bucle infinito Presionar botón STOP en IDE
