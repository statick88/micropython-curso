Laboratorio 3.1: Navegación con Giroscopio

⚡ Nivel: Intermedio-Avanzado | ⏱️ Tiempo: 45 minutos

Objetivo del Laboratorio

Implementar navegación de precisión utilizando el sensor giroscopio del Hub para lograr trayectorias rectas y giros exactos, indispensable para competencias de robot.

Fundamentos de Ingeniería

¿Por qué necesitamos el giroscopio?

Los motores LEGO, por sí solos, tienen juego mecánico (backlash). Esto significa que:

SIN Giroscopio:                    CON Giroscopio:
┌─────────────────────┐            ┌─────────────────────┐
│  Motor Izq ████     │            │  Motor Izq ████    │
│  Motor Der ████     │  ──────►   │  Motor Der ████    │
│                     │   Error:   │  + Gyro Feedback   │
│ Trayectoria:        │   15-20%   │  Trayectoria:      │
│ ════════════►       │            │  ════════════►     │
│                     │            │  Error: < 2%      │
└─────────────────────┘            └─────────────────────┘

Componente Especificación

Paso 2: Navegación Cuadrada (15 minutos)

from pybricks.pupdevices import Motor
from pybricks.parameters import Port, Direction
from pybricks.robotics import DriveBase
from pybricks.tools import wait

robot = DriveBase(Motor(Port.A, Direction.COUNTERCLOCKWISE), 
                 Motor(Port.B), wheel_diameter=56, axle_track=112)

def cuadrado_perfecto(lado_mm=300, repeticiones=1):
    """
    Navega un cuadrado perfecto usando control giroscópico.
    
    Args:
        lado_mm: Longitud de cada lado en milímetros
        repeticiones: Cuántas veces repetir el cuadrado
    """
    robot.use_gyro(True)
    robot.settings(straight_speed=200)
    
    for i in range(repeticiones):
        print(f">>> Iteración {i+1}/{repeticiones}")
        
        # Avanzar lado 1
        robot.straight(lado_mm)
        wait(500)
        
        # Girar 90 grados
        robot.turn(90)
        wait(500)
        
        # Avanzar lado 2
        robot.straight(lado_mm)
        wait(500)
        
        # Girar 90 grados
        robot.turn(90)
        wait(500)
        
        # Avanzar lado 3
        robot.straight(lado_mm)
        wait(500)
        
        # Girar 90 grados
        robot.turn(90)
        wait(500)
        
        # Avanzar lado 4 (regreso a origen)
        robot.straight(lado_mm)
        
        # Girar 90 grados para orientar como al inicio
        robot.turn(90)
        
        # Stop final
        robot.stop()
        
    print(">>> Cuadrado completado")

if __name__ == "__main__":
    cuadrado_perfecto(lado_mm=300, repeticiones=1)

def cuadrado_optimizado(lado_mm=300, repeticiones=1):
    """Versión optimizada con bucle."""
    
    robot.use_gyro(True)
    robot.settings(straight_speed=250)  # Más rápido
    
    for ciclo in range(repeticiones):
        for lado in range(4):  # 4 lados = 1 cuadrado
            robot.straight(lado_mm)
            robot.turn(90)
            wait(200)
        
        print(f">>> Cuadrado {ciclo+1} completado")
    
    robot.stop()

Paso 4: Telemetry en Tiempo Real (10 minutos)

def navegacion_telemetria():
    """Navegación con diagnóstico en tiempo real."""
    
    robot.use_gyro(True)
    robot.settings(straight_speed=150)
    
    # ─────────────────────────────────────────────
    # TELEMETRÍA: Monitoreo en tiempo real
    # ─────────────────────────────────────────────
    while True:
        # Lectura de sensores
        distancia = robot.distance()
        angulo = robot.angle()
        
        # Mostrar en consola (tu HUD de diagnóstico)
        print(f"Dist: {distancia}mm | Ángulo: {angulo}°")
        
        # Condición de parada
        if distancia >= 500:  # 50cm
            robot.stop()
            print(">>> Destino alcanzado")
            break
        
        wait(50)  # Update cada 50ms

if __name__ == "__main__":
    navegacion_telemetria()

🛠️ Centro de Diagnóstico

Error Síntoma Causa Solución
“Drifts” Robot se sale de línea Giroscopio no inicializado Verificar use_gyro(True)
“Giro incompleto” Gira 85° en vez de 90° Velocidad muy alta Reducir turn_speed
“Motores no responden” Robot no se mueve Puertos incorrectos Cambiar A↔︎B
“Zigzag” Movimiento ondulante ruedas desbalanceadas Verificar presión de neumáticos
“overshoot” Pasa de la línea Freno no configurado Usar robot.stop() inmediato
