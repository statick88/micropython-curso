⚠️ Nota sobre Licencias de Componentes

El firmware Pybricks incluye bibliotecas de terceros. La mayoría son open source, pero BTstack tiene una licencia más restrictiva que prohibe cobrar por el firmware.

🖥️ Requisitos por Plataforma

Ubuntu/Linux 24.04 (RECOMENDADO)

Herramienta Versión Instalación
Git Cualquiera reciente sudo apt install git
Python 3.12.x sudo apt install python3
Poetry 2.x pipx install poetry
ARM GCC Toolchain 13.x sudo apt install gcc-arm-none-eabi
GCC Sistema sudo apt install build-essential
Make Sistema sudo apt install make

Instalación completa:

sudo apt update
sudo apt install git python3 build-essential gcc-arm-none-eabi
pipx install poetry

Windows (vía Cygwin)

Herramienta Descripción
Cygwin Entorno Unix para Windows
ARM GCC Toolchain Compilador cross-compilation
Python 3.10+

Instalación paso a paso:

  1. Descargar e instalar Cygwin:

    • https://www.cygwin.com/setup-x86_64.exe
    • Paquetes a seleccionar:
      • doxygen
      • graphviz
      • make
      • mingw64-x86_64-gcc-core
      • python3
      • zip

  2. Instalar ARM GCC Toolchain:

    • https://developer.arm.com/downloads/-/arm-gnu-toolchain-downloads
    • Descargar versión 13.x para Windows

  3. Configurar PATH:

    $env:PATH="C:\cygwin64\bin;C:\Program Files (x86)\GNU Arm Embedded Toolchain\10 2020-q4-major\bin;$env:PATH"

macOS

Herramienta Instalación
Git Incluido o brew install git
Python brew install python
Poetry pipx install poetry
ARM GCC Toolchain brew install --cask gcc-arm-embedded
libusb brew install libusb

Instalación completa:

brew install pipx
pipx install poetry
brew install libusb
brew install --cask gcc-arm-embedded

📥 Descargar el Código Fuente

Opción 1: Línea de Comandos (RECOMENDADA)

# Clonar el repositorio (NO usar --recursive, es muy lento)
git clone https://github.com/pybricks/pybricks-micropython
cd pybricks-micropython

# Abrir en VS Code (opcional)
code .

Opción 2: GitHub Desktop

  1. Descargar: https://desktop.github.com
  2. Clone → https://github.com/pybricks/pybricks-micropython

Opción 3: VS Code

  1. Command Palette → Git: Clone
  2. Pegar URL del repositorio

⚙️ Configurar Entorno Python

# Verificar versión de Python (necesita 3.10+)
python3 --version

# Si tienes múltiples versiones, configurar Poetry
poetry env use python3

# Instalar dependencias
poetry install

# Activar entorno virtual
eval $(poetry env activate)

# Verificar que está activado (debe mostrar (.venv) al inicio)
# (.venv) $

🔨 Compilar el Firmware

Paso 1: Configurar Compilador ARM

# Opción A: Agregar al PATH
export PATH=$PATH:~/gcc-arm-none-eabi-*/bin

# Opción B: Usar variable CROSS_COMPILE
export CROSS_COMPILE=~/gcc-arm-none-eabi-*/bin/arm-none-eabi-

Paso 2: Compilar para tu Hub

# Activar entorno Poetry
eval $(poetry env activate)

# Compilar mpy-cross primero (necesario para el proceso)
make mpy-cross -j8

# Compilar firmware para SPIKE Prime Hub
make -C bricks/primehub -j8

# Otras opciones de Hubs:
make -C bricks/cityhub -j8      # LEGO City Hub
make -C bricks/technichub -j8   # LEGO Technic Hub
make -C bricks/boosthub -j8     # LEGO BOOST Hub
make -C bricks/inventorhub -j8  # MINDSTORMS Inventor

Paso 3: Encontrar el Firmware

Después de compilar, el firmware está en:

pybricks-micropython/
└── bricks/
    └── primehub/
        └── build/
            ├── firmware.bin      ← Firmware principal
            └── firmware-zip.zip  ← Para instalar

📲 Flashear el Firmware al Hub

Método 1: Pybricks Code (FÁCIL)

  1. Ir a https://pybricks.com/install
  2. Conectar Hub por USB
  3. Seguir instrucciones en pantalla
  4. Seleccionar tu archivo firmware.zip
  5. Click “Install”

Método 2: DFU Mode (AVANZADO)

# Para SPIKE Prime / MINDSTORMS Inventor
# 1. Poner Hub en modo DFU:
#    - Mantener CENTER + botón verde al encender

# 2. Flashear con dfu-util
dfu-util -a 0 -D firmware.bin

🧪 Probar el Firmware Instalado

Con Pybricks Code (Online)

  1. Abrir https://code.pybricks.com
  2. Conectar Hub por USB
  3. Click “Connect”
  4. Escribir código de prueba:
from pybricks.hubs import PrimeHub
from pybricks.parameters import Icon
from pybricks.tools import wait

hub = PrimeHub()

hub.display.show_icon(Icon.HEART)
wait(1000)
hub.display.show_icon(Icon.HAPPY)
hub.speaker.beep(440, 200)

print("¡Pybricks funciona!")
  1. Click ▶️ Run

🎓 Ejercicio para Estudiantes

Objetivo

Compilar firmware Pybricks desde código fuente.

Pasos

# 1. Clonar repositorio
git clone https://github.com/pybricks/pybricks-micropython
cd pybricks-micropython

# 2. Instalar Poetry (si no tienes)
pipx install poetry

# 3. Configurar entorno
poetry install
eval $(poetry env activate)

# 4. Instalar ARM GCC (Ubuntu)
sudo apt install gcc-arm-none-eabi

# 5. Compilar
make mpy-cross -j8
make -C bricks/primehub -j8

# 6. Verificar resultado
ls -la bricks/primehub/build/

💡 Tips de Compilación

┌─────────────────────────────────────────────────────────────────────────────┐
│                          💡 TIPS PROFESIONALES                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   1. 🐢 NO USES --recursive AL CLONAR                                      │
│      └─▶ Los submodules se cargan on-demand, es más rápido                 │
│                                                                             │
│   2. 🔧 USA -j8 PARA PARALELIZAR                                           │
│      └─▶ make -C bricks/primehub -j8 (usa 8 cores)                         │
│                                                                             │
│   3. 📦 PRIMERO COMPILA mpy-cross                                          │
│      └─▶ make mpy-cross -j8 (necesario para el proceso)                    │
│                                                                             │
│   4. 🐛 SI HAY ERRORES, LIMPIA PRIMERO                                     │
│      └─▶ make clean && make -C bricks/primehub -j8                         │
│                                                                             │
│   5. 📊 VERIFICA EL TAMAÑO DEL FIRMWARE                                    │
│      └─▶ ls -lh bricks/primehub/build/firmware.bin                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
