Fundamentos: Antes de Comenzar

Nuevo: Módulo de Fundamentos de Python

Si nunca has programado antes o quieres reforzar tus conocimientos de Python antes de trabajar con el hardware, te recomendamos comenzar con nuestro Módulo de Fundamentos de Python para Robótica Educativa.

Este módulo cubre: - Variables y tipos de datos - Estructuras de control (condicionales y bucles) - Funciones - Manejo de errores - Estructuras de datos (listas y diccionarios)

👉 Acceder al módulo de fundamentos

Algo interesante antes de empezar

Antes de escribir nuestro primer programa, vamos a entender algo que usaremos siempre: cómo guardar información que puede cambiar.

Imagina esto:Estás tomando asistencia en tu clase. Cada vez que entra un estudiante, cuentas “uno más”. Ese número cambia todo el tiempo - empieza en 0, luego es 1, luego 2, y así sucesivamente.

¿Cómo harías esto en un programa?

Podrías pensar: “Necesito algo que guarde el número actual y que pueda cambiarlo cuando entre otro estudiante”.

Eso es exactamente lo que aprenderemos aquí [1][2].

Una situación del día a día

Imagina que tienes una caja donde pones el conteo de estudiantes. Cuando entra alguien,.abres la caja, miras el número que hay dentro, le sumas 1, y lo vuelves a meter.

Caja tiene: 0
Entra un estudiante → Caja ahora tiene: 1
Entra otro estudiante → Caja ahora tiene: 2

En programación, a esa “caja” le llamamos de otra manera, pero la idea es exactamente la misma [1].

Primero, probemos en la computadora

Abre Thonny en tu computadora y escribe esto [3]:

# Esta "caja" se llama conteo
conteo = 0

# Cuando entra un estudiante, hacemos esto:
conteo = conteo + 1
print(conteo)  # Mostrará: 1

# Entra otro
conteo = conteo + 1
print(conteo)  # Mostrará: 2

Ejecuta el código (presiona F5 o el botón Play). Verás que el número cambia cada vez [3].

¿Qué está pasando? - conteo = 0 → Creamos la “caja” con valor inicial 0 - conteo = conteo + 1 → Sacamos el valor, le inmuevo1, lo volvemos a meter

Ahora, en el robot

Cuando programamos el LEGO Hub, la idea es igual, pero la syntax es un poco diferente [4][5]:

# En el robot, usamos el mismo patrón
conteo_estudiantes = 0

# Simular que entra un estudiante
conteo_estudiantes = conteo_estudiantes + 1

# Mostrar en la pantalla del robot
hub.display.number(conteo_estudiantes)

La diferencia es solo en cómo mostramos el resultado - en la pantalla del robot en lugar de la consola.

Diferentes tipos de “cajas”

En el ejemplo anterior solo usamos números, pero a veces necesitamos guardar otras cosas [2]:

Qué quieres guardar Ejemplo
Un número que puede crecer contador = 5
Un número con decimales temperatura = 22.5
Un nombre o palabra nombre = "Ana"
Una respuesta de sí/no presente = True

Imagina tener una receta escrita:

RECETA PARA SALUDAR:
1. Decir "Hola"
2. Decir el nombre que me den

Ahora solo escribimos:

saludar("Ana")
saludar("Carlos")
saludar("María")

Mucho más fácil, ¿verdad?

Probemos esta “receta” en la computadora

En Thonny, escribe [3]:

def saludar(nombre):
    print("Hola " + nombre)

# Ahora podemos usar la receta muchas veces
saludar("Ana")
saludar("Carlos")
saludar("María")

Ejecuta y verás cómo con una sola “receta” saludamos a todos.

Y en el robot

Con el LEGO Hub, podemos crear “recetas” igual [4][5]:

def mostrar_mensaje(texto):
    """Esta receta muestra un mensaje en la pantalla"""
    hub.display.text(texto)

# Usar la receta
mostrar_mensaje("BIENVENIDOS")
mostrar_mensaje("HOLA")

Un último concepto: repetir sin escribir muchas veces

Imagina que el director te pide: “Dile a todos los estudiantes que hay examen mañana”.

Podrías: - Llamar a cada estudiante individualmente (mucho trabajo) - Hacer un anuncio por megafonía (una sola vez, todos escuchan)

En programación, cuando queremos que algo se haga varias veces, usamos algo llamado bucle [6][7].

Un bucle en la computadora

# Saludar a 5 estudiantes
for i in range(5):
    print("Hola estudiante #" + str(i + 1))

Esto mostrará:

Hola estudiante #1
Hola estudiante #2
Hola estudiante #3
Hola estudiante #4
Hola estudiante #5

Solo escribimos el código una vez, pero se ejecuta 5 veces [6].

Y en el robot

# El robot avanza 4 veces (como un cuadrado)
for lado in range(4):
    robot.straight(300)  # Avanzar
    robot.turn(90)       # Girar

El robot hará: adelante → gira → adelante → gira → adelante → gira → adelante → gira

Resumen de lo que vimos

Concepto Analogía Para qué sirve
“Caja” Contenedor para guardar algo que cambia Guardar valores que necesitamos recordar [1][2]
“Receta” Instrucciones que podemos reutilizar No escribir lo mismo varias veces [4]
“Bucle” Hacer algo varias veces Repetir acciones sin copiar y pegar [6][7]

Estos tres conceptos los usaremos constantemente en todo el curso. Ahora que los entiendes (a través de ejemplos de la vida real), podemos aplicarlos al robot.

Otro concepto: hacer cosas sin repetir

Ahora imagínate esto: Tienes que saludar a cada estudiante por su nombre cuando entran. Podrías escribir:

print("Hola Ana")
print("Hola Carlos")
print("Hola María")
# ...y así sucesivamente

Pero eso es muy largo. ¿No sería mejor tener una sola “instrucción” que podamos usar una y otra vez? [4]
