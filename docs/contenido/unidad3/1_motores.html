Unidad 3: ¿Cómo hace el robot para moverse?

⏱️ Tiempo estimado: 50 minutos | 📊 Nivel: Intermedio

Antes de empezar

¿Alguna vez has observado cómo funciona un carro de juguete? Tienes dos ruedas, y cuando ambasavan, el carro avanza. Cuando una gira más que la otra, el carro gira.

El robot LEGO funciona exactamente igual—tiene dos “rueditas” (motores) que puede controlar independientemente. Vamos a aprender cómo hacerlo [1][2].

Las dos ruedas del robot

Situación: Estás en un supermercado con un carrito. Si empujas las dos ruedas igual, el carrito va recto. Si empujas más la rueda derecha, el carrito gira a la izquierda.

El robot tiene algo parecido—dos motores que pueden ir a diferentes velocidades o en diferentes direcciones [2].

┌─────────────────────────────────────┐
│           ROBOT (visto desde arriba)  │
│                                       │
│    ┌─────────┐        ┌─────────┐     │
│    │  MOTOR  │ ←────→ │  MOTOR  │     │
│    │   A     │        │   B     │     │
│    └─────────┘        └─────────┘     │
│         ↑                   ↑         │
│    Rueda izq          Rueda der       │
└─────────────────────────────────────┘

¿Para qué sirve? - Ambas van igual → el robot va recto - Una va, la otra no → el robot gira - Una va hacia adelante, otra hacia atrás → el robot gira más rápido

Antes de dar órdenes al robot

Situación: Antes de manejar un carro nuevo, primero necesitas asegurarte de que todo está bien—tienes que calibrarlo, ajustar los espejos, etc.

Antes de pedirle al robot que se mueva, necesitamos “presentarle” los motores [2]:

# Traer las herramientas necesarias
from pybricks.pupdevices import Motor
from pybricks.parameters import Port
from pybricks.robotics import DriveBase

# Cada motor se conecta a un "puerto" (como un enchufe)
# Puerto A = motor izquierdo
# Puerto B = motor derecho
motor_izquierdo = Motor(Port.A)
motor_derecho = Motor(Port.B)

# Ahora creamos algo que controle el movimiento
# Le decimo el tamaño de las ruedas (56mm) y qué tan separadas están (112mm)
robot = DriveBase(motor_izquierdo, motor_derecho, wheel_diameter=56, axle_track=112)

¿Por qué necesitamos esas medidas? - Es como cuando en el auto le dices al GPS el tamaño de las llantas—así sabe cuántos metros ha avanzado - Sin esas medidas, el robot no sabría cuánto es “un metro”

¡Ahora el robot se mueve!

Una vez que está “presentado”, podemos darle órdenes [2]:

# "¡Robot, avanza 50 centímetros!" (500 milímetros)
robot.straight(500)

# "¡Gira 90 grados a la derecha!"
robot.turn(90)

# "¡Gira 90 grados a la izquierda!" (número negativo)
robot.turn(-90)

# "¡Detente!"
robot.stop()

Practica esto en Thonny:

from pybricks.pupdevices import Motor
from pybricks.parameters import Port
from pybricks.robotics import DriveBase
from pybricks.tools import wait

# Configurar (presentar) los motores
motor_izq = Motor(Port.A)
motor_der = Motor(Port.B)
robot = DriveBase(motor_izq, motor_der, wheel_diameter=56, axle_track=112)

# ¡Probar el movimiento!
robot.straight(200)  # Avanzar 20 cm
wait(500)            # Pausa
robot.turn(90)       # Girar 90°
wait(500)
robot.straight(200)  # Avanzar otro poco
wait(500)
robot.stop()         # Detenerse

El equilibrio del robot (el giroscopio)

Situación: Cuando caminas con los ojos vendados, te desvías. Pero si miras un punto fijo, puedes caminar recto. El robot tiene algo similar—un “sentido del equilibrio” que lo ayuda a ir derecho.

Ese “sentido” se llama giroscopio. Ayuda al robot a no desviarse cuando avanza mucho [2].

# Activar el "sentido del equilibrio"
robot.use_gyro(True)

# Ahora el robot caminará recto incluso en distancias largas
robot.straight(1000)  # 1 metro recto

¿Cuándo usarlo? - Cuando el robot necesita ir muy recto (como una línea) - Cuando necesita girar exacto (como 90°) - Cuando el piso no es parejo

Hacer que el robot repita algo

Situación: ¿Y si quieres que el robot haga lo mismo 4 veces? ¿Escribes el código 4 veces?

Hay una forma mejor. Es como cuando en el recreo haces “una fila de 4” y todos repiten lo mismo [3].

# "Haz esto 4 veces"
for i in range(4):
    robot.straight(200)  # Avanzar
    robot.turn(90)       # Girar
    # y vuelve al inicio del "for"

El robot entiende “for i in range(4)” como “haz lo que está dentro 4 veces”.

Ejemplo: Un cuadrado

# El robot va a dibujar un cuadrado (4 lados iguales)
for i in range(4):
    robot.straight(300)  # Un lado de 30 cm
    robot.turn(90)      # Girar 90°
    
robot.stop()  # ¡Terminó!

¿Qué pasa si quiero algo más preciso?

A veces el robot se desvía un poco—como cuando manejas en una calle dispareja. Pero podemos ajustar [2]:

# Ajustar la velocidad máxima
robot.settings(straight_speed=150)  # Más lento = más preciso

# O ajustar la fuerza de giro
robot.settings(turn_rate=45)  # Girar más despacio

Es como manejar un carro—si vas más lento, es más fácil hacer curvas exactas.
