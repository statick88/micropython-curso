📝 Preguntas

🐍 Pregunta 1: Variables (25 XP)

En robótica, ¿para qué sirve principalmente una variable?

💡 Pista: Las variables son como las “cajas de herramientas” del robot - guardan valores que pueden cambiar mientras el programa se ejecuta [1].

🔢 Pregunta 2: Tipos de Datos (25 XP)

Si tu robot necesita medir la temperatura ambiente y mostrarla, ¿qué tipo de dato usarías?

💡 Pista: La temperatura como 22.5°C necesita decimales, no un número entero [2].

🔀 Pregunta 3: Condicionales (25 XP)

¿Qué estructura usarías para que el robot “decida” si avanzar o detenerse según la distancia a un obstáculo?

💡 Pista: Los condicionales if/else permiten que el programa tome diferentes caminos según una condición - como decidir si avanzar o detenerse [3].

🔁 Pregunta 4: Bucles (25 XP)

Tu robot necesita avanzar exactamente 4 veces para formar un cuadrado. ¿Qué estructura es la más apropiada?

💡 Pista: Cuando sabes cuántas veces necesitas repetir, for es la mejor opción - como dar 4 pasos para un cuadrado [4].

📝 Pregunta 5: Funciones (25 XP)

¿Por qué es mejor usar funciones en nuestro código de robótica?

💡 Pista: Las funciones son como “recetas” - las escribes una vez y las usas cuando las necesites [5].

⚠️ Pregunta 6: Manejo de Errores (25 XP)

Si el sensor de distancia del robot deja de responder, ¿qué estructura nos ayuda a manejar ese error elegantemente?

💡 Pista: El bloque try/except permite que nuestro programa no falle catastróficamente cuando algo sale mal - como cuando un sensor no responde [6].

📦 Pregunta 7: Listas vs Diccionarios (25 XP)

Para guardar el historial de 5 lecturas del sensor de distancia, ¿qué estructura es mejor?

💡 Pista: Las listas son perfectas para colecciones ordenadas del mismo tipo de datos - como lecturas sucesivas de un sensor [7].
