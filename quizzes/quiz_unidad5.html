📝 Preguntas Teóricas

🧠 Pregunta 1: Integración de Sistemas (25 XP)

¿Qué componente es EL CENTRO DE CONTROL que orchestra todas las funciones del robot en Pybricks?

💡 Pista: El Hub es el cerebro central que gestiona LEDs, botones, speaker, IMU y comunicación [1]. ¡Sin cerebro no hay robot! 🧠

📐 Pregunta 2: Cinemática y Giroscopio (25 XP)

Para lograr que un robot recorra una línea recta perfecta, ¿qué configuración es OBLIGATORIA?

💡 Pista: Sin el giroscopio, el robot drifteará 15-20% de error en trayectoria [2]. ¡Sin giro = borracho robot! 🍺

⚡ Pregunta 3: Programación Asíncrona (25 XP)

En un sistema de navegación autónoma, ¿cuál es el propósito de await wait(50) dentro de un loop async?

💡 Pista: El await wait() permite que el scheduler de tareas cambie a otras corrutinas [3]. ¡Multitasking real! 🤹

💡 Pregunta 4: Matriz de LEDs (25 XP)

Si deseas mostrar una animación en la matriz 5x5, ¿cómo se representa cada píxel?

💡 Pista: Cada LED se controla mediante una matriz de enteros de 0 (apagado) a 100 (máximo brillo) [4]. ¡Tu canvas digital! 🎨

🏗️ Pregunta 5: Arquitectura de Código (25 XP)

¿Cuál es el enfoque correcto para escribir código en sistemas embebidos?

💡 Pista: Las funciones bien definidas permiten diagnóstico, testing y reutilización [5]. ¡Clean Code es Cool Code! 😎

🎯 Pregunta 6: Diseño de Sistemas (25 XP)

¿Cuál es el PRIMER paso en la metodología Statick antes de escribir código?

💡 Pista: “No estamos aquí para enseñar sintaxis; estamos aquí para diseñar sistemas” [6]. ¡Planificar es Vencer! 🏆

🔧 Pregunta 7: Resolución de Problemas (25 XP)

Tu robot no responde. ¿Cuál es el PRIMER paso para diagnosticar?

💡 Pista: El diagnóstico clínico siempre comienza por verificar las conexiones y energía [7]. ¡KISS: Keep It Simple, Stupid! 🤓

💬 Pregunta 8: Feedback Visual (25 XP)

¿Cuál es la mejor manera de comunicar el estado del sistema al usuario?

💡 Pista: En sistemas embebidos, el feedback visual/sonoro es la única interfaz con el usuario [1]. ¡Tu robot debe hablar! 🗣️
